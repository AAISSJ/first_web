자바에서 사용하는 용어 중 혼동하기 쉬운 JVM, JRE, JDK에 대해서 정리해 보자!
JVM (Java Virtual Machine)은 자바 소스코드로부터 만들어지는 자바 바이너리 파일(.class)을 실행할 수 있다. 또한 JVM은 플랫폼에 의존적이다. 즉 리눅스의 JVM과 윈도우즈의 JVM은 다르다. 단, 컴파일된 바이너리 코드는 어떤 JVM에서도 동작시킬 수 있다.
RE는 자바 실행환경(Java Runtime Environment)의 약자이다. JRE는 JVM 이 자바 프로그램을 동작시킬 때 필요한 라이브러리 파일들과 기타 파일들을 가지고 있다. JRE는 JVM의 실행환경을 구현했다고 할 수 있다.
JDK는 자바 개발도구(Java Development Kit)의 약자이다. JDK는 JRE + 개발을 위해 필요한 도구(javac, java등)들을 포함한다.

위의 jvm을 어디서 많이 봤다 했더니 2학년 1학기 프로젝트로 코틀린을 했을 때 본 것이다! 코틀린은 자바와 100% 호환이 가능한 언어로, 호환이 가능한 이유는 바로 jvm 때문이다! jvm은 Compilation과 Interpretation의 조합이라고 할 수 있다.

  1. JavaC Compiler 에 의한 Compilation : [Java Code -> Byte code (.class)]
  2. Interpretating : [Byte code -> 각각의 운영 체제에 맞게 (MAC, 윈도우, 리눅스)]

그리고 이 Compilation과 Interpretation을 구분해보자!

  1. Compilation (C, C++, GO, Fortron ...)
    : Language->Machine Code->Ready to Run
    :Interpretation보다 빠르나, 플랫폼에 의존적임 (MAC, 리눅스, 윈도우로 컴파일하려면 각각 써야)
  2. Interpretation (PYTHON, PHP, JS ...)
    :Language->Ready to Run->Virtual Machine->Machine Code
    :interpreter가 알아서 MAC, 윈도우, 리눅스 등으로 변환

ex) String id=JOptionPane.InputDialog(" ")하면 알아서 필요한 라이브러리를 import함.

이클립스에선 실행 키 밑에 있는 run configurations에 들어가 현재 java 파일을 누르면 Program argumenets에다가 입력값 한꺼번에 넣을 수 있음! 단, 띄어쓰기 단위로 input값이 나뉘기 때문에 띄어쓰기 포함하여 하나로 입력하려면 " "로 묶어야 한다. Apply 누르고 실행하면 끝! 여러 가지 Test Set 하고 싶으면 이름 다르게 저장할 수 있다. 또 Program argumenets에 입력한 값들은 소스 코드에서 args[0], args[1] 등으로 가리킬 수 있는데, 이는 매개변수(parameter) args에다가 argumenets를 넣은 것이기 때문에 그렇다.

Java 첫째 날에 썼듯이, Java는 java 확장자가 붙은 소스코드가 class 확장자가 붙은 파일로 바뀐다.(compile) 그리고 그 class 파일을 실행하면 프로그램이 돌아간다.(run)

이클립스 없이 직접 compile 하기!

  • 시스템 환경 변수-사용자 변수에 있는 path에 java.exe가 있는 경로 입력
  • cmd창에서 cd (실행하고자 하는 hello.java 파일이 있는 디렉토리 경로) 입력하여 이동
  • javac hello.java 하면 hello.class로 compile 가능/ 만약 import한 라이브러리가 같은 디렉토리 내에 없는 파일이라면? (그 대신 lib 디렉토리에 있다 치자) javac -cp ".;lib" hello.java 라고 입력하면 된다. 단, 윈도우는 ; 다른 운영체제는 :로 바꿔준다.

    저 문서에서 class PrintWriter를 검색해보면
    java.lang.object
    -java.io.Writer
    --java.io.PrintWriter
    위와 같이 되어 있다. 세번째 클래스는 두번째 클래스를 상속받은 것, 두번째 클래스는 첫번째 클래스를 상속받은 것이다. 따라서 세번째 클래스에서 선언하지 않고 첫번째에서 선언한 toString과 같은 메소드 등을 사용할 수 있다! 세번째 클래스의 toString은 두번째 클래스의 toString을, 두번째 클래스의 toString은 첫번째 클래스의 toString을 "Override"헸다!
